SGDC sans document.write

Les utilisateurs ont la possibilité d'implémenter SGDC sans avoir à utiliser document.write. Cet exemple de page est destiné aux utilisateurs qui utilisent actuellement SGDC et souhaitent convertir leur application à la nouvelle méthode qui n'utilise pas document.write.

Mise en œuvre

Références de fichier de la section <head>

Ajouter un link vers la feuille de style de SGDC appropriée pour votre cas. Une des suivantes:

  • cdts-styles.css: Le gabarit GCWeb de base
  • cdts-app-styles.css: Le gabarit GCWeb application
  • cdts-splash-styles.css: Le gabarit GCWeb page d'entrée

2. Ajouter l'élément script référençant le fichier Javascript de SGDC (wet-[language].js où [langage] est soit "en" ou "fr"). Le SGDC est activé soit en spécifiant un attribut data-cdts-setup contenant l'objet JSON de configuration de SGDC.

3. Créer une section <noscript> ce qui permettra à votre page d’avoir la présentation voulue si JavaScript est désactivé. Vous devrez charger un fichier statique de repli dans la section <noscript>.

L'objet the configuration SGDC qui est passé dans l'attribut data-cdts-setup doit inclure les propriétés suivantes :

  • cdnEnv (FACULTATIF, par défaut "prod") - Chaîne de caractères indiquant l'environnement SGDC à utiliser pour télécharger les ressources. Valeurs possibles:
    • Environnement hôte: https://www.canada.ca
      "cdnEnv": "prod"
    • Environnement hôte: https://cdts.service.canada.ca
      "cdnEnv": "esdcprod"
  • mode (FACULTATIF, par défaut "common") - Chaîne de caractères indiquant quel mode de gabarit à utiliser. Valeurs possibles:
    • Le gabarit GCWeb de base
      "mode": "common"
    • Le gabarit GCWeb application (doit aussi s'assurer que le fichier CSS correspondant cdts-app-styles.css est utilisé)
      "mode": "app"
    • Le gabarit GCWeb server (utilisé typiquement pour les pages d'erreur)
      "mode": "server"
    • Le gabarit GCWeb splash (doit aussi s'assurer que le fichier CSS correspondant cdts-splash-styles.css est utilisé)
      "mode": "splash"
  • base (FACULTATIF) - Les configurations refTop et refFooter vont ici.
  • top, preFooter, footer, secmenu, splash (FACULTATIF) - Objets indiquant la configuration de chaque section, au besoin.
<link rel="stylesheet" href="../cdts-styles.css">
<script type="text/javascript" src="../compiled/wet-en.js" data-cdts-setup='{"cdnEnv": "esdcprod","base":{"exitSecureSite":{"exitScript":true,"displayModal":true,"exitURL":"http://www.google.ca","exitDomains":"www.esdc.gc.ca, www.jobbank.gc.ca, www.redseal.ca","displayModalForNewWindow":true}},"top":{"lngLinks":[{"href":"nodoc-write-fr.html","lang":"fr","text":"Français"}]},"preFooter":{},"footer":{},"onCDTSPageFinalized":[]}'></script>

Références de fichier de la section <body>

Ajoutez trois éléments div avec les ID mentionnés ci-dessous.

Insérez/incluez du contenu statique de secours dans le nouveau div que vous avez créé.

  • Ajoutez un nouvel élément div avec un ID de cdts-def-top. Il doit être situé avant l'élément main sur la page.
  • Ajoutez un nouvel élément div avec un ID de cdts-def-preFooter. Il doit être situé à l'intérieur de l'élément main sur la page.
  • Ajoutez un nouvel élément div avec un ID de cdts-def-footer. Il doit être situé après l'élément main sur la page.
<body vocab="https://schema.org/" typeof="WebPage">
    <div id="cdts-def-top"></div>
    <main property="mainContentOfPage" typeof="WebPageElement" class="container">
        ...
        <div id="cdts-def-preFooter">
        </div>
    </main>
    <div id="cdts-def-footer"></div>
</body>
