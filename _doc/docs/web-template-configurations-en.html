CDTS Web Template Configurations

Slogan

Note

The Web Template offers multiple ways to configure your site and its behavior to meet your needs. Below are the items that may be configured either statically via the configuration entries (web.config for .NET or goc.webtemplate.global.config.cdn.properties for Java) or programmatically. The code samples for each of these can be found in the Zip files of the web templates and also on NuGet for ESDC .Net team.

Web Template Configurations

Environment

This setting determines which CDTS environment you will use to consume the web assets.

  • Set in the config:
    • GoC.WebTemplate.Environment:
      • Specifies the CDTS to use
      • 6 values are available:
        • AKAMAI:
          • GCWeb Theme:
            • GCWeb is the only theme supported in this environment
            • To be used by all public facing applications.
            • The same environment is to be used for your development, testing and production implementation
        • PROD_SSL
          • GCIntranet Theme is the only theme supported in this environment
            • To be used by all GoC Intranet facing applications
        • PROD_UNSECURE
          • GCIntranet Theme is the only theme supported in this environment
            • To be used by all GoC Intranet facing applications
        • ESDC_PROD:
          • GCIntranet Theme is the only theme supported in this environment
            • To be used by all ESDC internal facing applications.
            • To be used for your production implementation only.
            • Use "ESDCNonProd" for your development and testing
        • SADE Environments:
          • ESDC_GCWEB_SADE
            • This CDTS will soon no longer support this theme. It is currently available strictly to support the migration to the Akamai CDTS.
            • GCWeb consumers should use the "Akamai" environment
          • ESDC_GCIntranet_SADE
            • To be used by ESDC internal facing applications.
            • This CDTS is not available to external departments.
            • To be used for your development and testing implementation only.
            • Use "ESDCProd" for your production implementation
        • ESDC QA Environments
          • ESDC_GCIntranet_QA
            • This CDTS will soon no longer support this theme. It is currently available strictly to support the migration to the Akamai CDTS.
            • GCWeb consumers should use the "Akamai" environment
          • ESDC_GCWEB_QA
            • To be used by ESDC internal facing applications.
            • This CDTS is not available to external departments.
            • Can be used for your QA testing.

Load JQuery from Google

This setting determines if the JQuery files should be retrieved from Google or from the CDTS specified in the Environment setting of the Web Template (Akamai, ESDCProd etc...).

  • Set in the config:
    • GoC.WebTemplate.loadJQueryFromGoogle
    • Values:
      • true: will retreive the files from google
      • false: will retreive the files from the CDTS specified in the Environment setting.
  • Can also be set programmatically via the property "LoadJQueryFromGoogle" of the Web Template

SubTheme

This setting will determine which sub theme of the web assets to use from the CDTS. Example: The GCIntranet theme supports the "GC" theme and the "esdc" theme.

  • Set in the config:
    • GoC.WebTemplate.SubTheme
    • Values:
      • blank (empty value). Is the default
      • esdc
  • Can also be set programmatically via the property "WebTemplateSubTheme" of the Web Template

Use HTTPS

This can only be set if the IsSSLModifiable option is set in the CDTSEnvironments.json file otherwise must be null or not set in config

This setting determines if the communication between the browser and the CDTS should be encrypted.

  • Set in the config:
    • GoC.WebTemplate.useHTTPS
    • Values:
      • true: will use https
      • false: will use https
      • empty: https not available
  • Can also be set programmatically via the property "UseHTTPS" of the Web Template

Version

This setting will determine which version of the web assets to use from the CDTS. Changes to the web assets hosted on the CDTS will generate a new version of them. The versioning allows the CDTS to deploy changes asap and allows the applications flexibility to move to the latest version at their earliest convenience. The template can also support the "run" version of the CDTS web assets, this would mean you are not using the versioned files but are using the latest files as they are updated.

  • Set in the config:
    • GoC.WebTemplate.Version
    • ex: v4_0_17
    • leave empty to use the latest CDTS files and not the versioned files. You will also need to supply the "run" static fallback files. [see fallback section for details].
  • Can also be set programmatically via the property "WebTemplateVersion" of the Web Template

Header Configurations

Application Title

When using the "GCIntranet" theme in your application, you can optionally set the Application Title text and its URL.The application title is displayed at the top of the page, above the top menu.

  • Set programmatically via the "ApplicationTitle_Text" property of the Web Template
  • The URL for the application title is optional. If left blank the subTheme will determine the URL to use. You can override the default URL value of the subTheme by setting programmatically the "ApplicationTitle_URL" property of the Web Template.

Breadcrumbs

The Breadcrumbs are a list of links displayed near to the top of the content to help the user determine where they are on the site and to help them navigate. In the transactional mode, the Breadcrumb trail should be replaced with a link to exit the transaction, which directs the user to a safe point where they can abandon the current transaction and restart it should they wish.

  • Set programmatically by populating the "Breadcrumbs" collection of the Web Template. The "Breadcrumbs" has 3 properties:
    • "href": the url of the link. If left empty, the item will be displayed as text and not as a hyperlink;
    • "title": the text displayed on screen;
    • "acronym": if your title has an acronym, you can use this property to provide the full text of the title. It will be displayed when the user hovers over the link.

Header Title

This setting determines the title for your site by populating the <title> tag.

  • Set programmatically via the "HeaderTitle" property of the Web Template.

MetaTags

The Web Template provides a "string" collection to include HTML Elements, including Meta Tags, to the head section of the page. This collection can be used by your application to include any Meta Tag required by your site.

  • Set programmatically by populating the "HTMLHeaderElements" collection of the Web Template.
  • The collection is of type "String", and the complete html tag for your MetaTag to include should be supplied. ex: "<meta name="singer" content="Elvis">"

Pre Content

This will be used by Portfolio Web to insert content into the pre content space of your page. By default, this is ALWAYS shown on all pages. You will need authorization from Portfolio Web or the Principal Publisher to not include this content in your web asset.

  • Set in the config:
    • GoC.WebTemplate.ShowPreContent:
    • Values
      • true (displayed);
      • false (not displayed);
  • Can also be set programmatically via the property "ShowPreContent" of the Web Template.

Search

The Search is the GoC search control that is displayed at the top of the page. By default the search is ALWAYS shown on all pages (except for the Transactional Mode). You will need authorization from Portfolio Web or the Principal Publisher to not include this content in your application.

  • Set in the config:
    • GoC.WebTemplate.ShowFeatures:
    • Values:
      • true (displayed);
      • false (not displayed).

Body Configurations

Left Menu

Your application can display a left menu to help with navigation by selecting the proper master page for your content pages. The items of the menu are controlled programmatically and 3 levels of menu items are supported.

  • Set programmatically by populating the "LeftMenuItems" collection of "MenuSection" of the Web Template. The "MenuSection" has 3 properties:
    • Link: the link of the header of the menu section
    • Name: the name of the header of the menu section
    • OpenInNewWindow: default is false, but if set to true the link will open in a new window.
    • List of Link: represents the links for the menu section.
  • When "OpenInNewWindow" is set to true, a span tag will be included for accessibility and to notify the user that the link will open a new window.

Footer Configurations

About Links

The About us links in the footer of the page are pre-determined by the theme used for your site. Some themes do allow them to be overwritten with your own list. If you decide to provide your list, the pre-determined links from the theme will not be displayed.

Note: Not applicable to all themes.

  • Set programmatically by populating the "AboutLinks" collection of the Web Template. This collection is of type "Link", which has 2 properties:
    • "href": the url of the link;
    • "text": the text displayed on screen;

Contact Links

The "Contact US" links in the footer of the page are pre-determined by the theme. Some themes do allow them to be overwritten with your own list. If you decide to provide your list, the pre-determined links from the theme will not be displayed.

Note: Not applicable to all themes.

  • Set programmatically by populating the "ContactLinks" collection of the Web Template. This collection is of type "Link", which has 2 properties:
    • "href": the url of the link;
    • "text": the text displayed on screen.

DateModified

The DateModified is displayed near the bottom of the page. This element shares the same location as "VersionIdentifier" and only one of the two can be displayed at once. If a "DateModified" is supplied the date will take precedence over the "VersionIdentifier".

Set programmatically via the "DateModified" property of the Web Template.

Features

The Features are the GoC activities and initiatives that are displayed at the bottom of the content. By default the features are ALWAYS shown on all pages. You will need authorization from Portfolio Web or the Principal Publisher to not include this content in your web asset.

  • Set in the config:
    • GoC.WebTemplate.ShowFeatures:
    • Values:
      • true (displayed);
      • false (not displayed).

Post Content

This will be used by Portfolio Web to insert content into the post content space of your page. By default this is ALWAYS shown on all pages. You will need authorization from Portfolio Web or the Principal Publisher to not include this content in your web asset.

  • Set in the config:
    • GoC.WebTemplate.ShowPostContent:
    • Values:
      • true (displayed);
      • false (not displayed);
  • Can also be set programmatically via the property "ShowPostContent" of the Web Template.

Screen Identifier

The Screen Identifier is displayed near the bottom of the page. This element serves a unique identifier for your page and can be used to help communication between users and the service desk/support team to identify the exact location of a user in your application.

  • Set programmatically via the "ScreenIdentifier" property of the Web Template.

Version Identifier

The Version Identifier is displayed near the bottom of the page. This element shares the same location as "DateModified" and only one of the two can be displayed at once. If a "DateModified" is supplied the date will take precedence over the "VersionIdentifier".

  • Set programmatically via the "VersionIdentifier" property of the Web Template.

  • Set programmatically via the "PrivacyLink_URL" property of the web Template.

GCWeb Theme Specific Configurations

"Feedback" - "Report a Problem" Link

The Report a Problem Link (previously the "Feedback" Link) is a way for the users of your site to provide feedback on their experience navigating through the site or to report an issue. Displaying the link is optional but it cannot be contextualized and Portfolio Web or the Principal Publisher should be contacted to help you determine if it should be displayed on your site. The URL of the link is pre-defined by the theme but can be modified in the web.config or programmatically.

  • Set in the config:
    • GoC.WebTemplate.ShowFeedbackLink:
    • Values:
      • true (displayed);
      • false (not displayed);
  • Can also be set programmatically via the property "ShowFeedbackLink" of the Web Template.
  • If you wish to redirect the user to your own page instead of the default canada.ca page, set the config"FeedbackLinkURL" in the web.config to the url of your page, or set the property "FeedbackLinkURL" programmatically.

"Share this Page" Link:

The "Share This Page" Link is a way for the users to share the URL of the site via Social Media . Displaying the link is optional and Portfolio Web or the Principal Publisher should be contacted to help you determine if it should be displayed on your site and which Social Media site should be present.

  • Set in the config:
    • GoC.WebTemplate.ShowSharePageLink:
    • Values:
      • true (displayed);
      • false (not displayed);
  • Can also be set programmatically via the property "ShowSharePageLink" of the Web Template.

If you decide to show the link, you must provide the list of Social Media sites to be displayed to the user by programmatically populating the "SharePageMediaSites" collection of the Web Template. If the collection "SharePageMediaSites" is left empty, the link will not be displayed. The collection is of type "SocialMediaSites" which is an enum of the Web Template.
