CDTS without document.write

Users have the ability to implement CDTS without having to use document.write. This sample page is intended for users who currently use CDTS and would like to convert their application to the new method that doesn't use document.write.

Implementation

Section <head> references

1. Add a link to the CDTS style sheet appropriate to your situation. One of:

  • cdts-styles.css: The GCIntranet basic template
  • cdts-eccc-styles.css: The GCIntranet ECCC template
  • cdts-esdc-styles.css: The GCIntranet ESDC template
  • cdts-splash-styles.css: The GCIntranet splash template

2. Add the script element referencing the CDTS JavaScript file (wet-[language].js where [language] is either "en" or "fr"). CDTS is activated by specifying a data-cdts-setup attribute containing the CDTS configuration JSON object.

3. Create a <noscript> section which will make your page look good if JavaScript is disabled. You will need to load a fall-back static file in the <noscript> section.

The CDTS configuration object that is passed in the data-cdts-setup attribute should include the following properties:

  • cdnEnv (OPTIONAL, defaults to "prod") - A string specifying the CDTS environment from which to load resources. One of:
    • Hosting Environment: https://www.canada.ca
      "cdnEnv": "prod"
    • Hosting Environment: https://cdts.service.canada.ca
      "cdnEnv": "esdcprod"
  • mode (OPTIONAL, defaults to "common") - A string specifying which template mode to use. One of:
    • The GCIntranet basic template
      "mode": "common"
    • The GCIntranet application template
      "mode": "app"
    • The GCIntranet server template (typically used for error pages)
      "mode": "server"
    • The GCIntranet splash template (also must make sure the corresponding CSS cdts-splash-styles.css is used)
      "mode": "splash"
  • base (OPTIONAL) - refTop and refFooter configurations go here, as well as the sub-theme.
    • subTheme (OPTIONAL, defaults to null/base theme) - A string specifying which sub-theme to use. If specified, possible values are esdc, eccc or labour (also must make sure the corresponding CSS cdts-[subTheme]-styles.css is used).
  • top, preFooter, footer, secmenu, splash (OPTIONAL) - Objects specifying the configuration of each section as needed. Refer to the corresponding section documentation below for details.
<link rel="stylesheet" href="../cdts-styles.css">
<script type="text/javascript" src="../compiled/wet-en.js" data-cdts-setup='{"cdnEnv": "esdcprod","base":{"exitSecureSite":{"exitScript":true,"displayModal":true,"exitURL":"http://www.google.ca","exitDomains":"www.esdc.gc.ca, www.jobbank.gc.ca, www.redseal.ca","displayModalForNewWindow":true}},"top":{"lngLinks":[{"href":"nodoc-write-fr.html","lang":"fr","text":"Français"}]},"preFooter":{},"footer":{},"onCDTSPageFinalized":[]}'></script>

Section <body> references

Add three div elements with IDs mentioned below.

Insert/include fall-back static content in the new div you have created.

  • Add a new div element with an ID of cdts-def-top. It should be located before the main element on the page.
  • Add a new div element with an ID of cdts-def-preFooter. It should be located inside the main element on the page.
  • Add a new div element with an ID of cdts-def-footer. It should be located after the main element on the page.
<body vocab="https://schema.org/" typeof="WebPage">
    <div id="cdts-def-top"></div>
    <main property="mainContentOfPage" typeof="WebPageElement" class="container">
        ...
        <div id="cdts-def-preFooter">
        </div>
    </main>
    <div id="cdts-def-footer"></div>
</body>
